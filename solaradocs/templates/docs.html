Documentation

Learn how to use Solara Docs to create, manage, and collaborate on documentation with your team.

Introduction

Solara Docs is a modern documentation platform built for teams who need to create, organize, and maintain documentation efficiently. With real-time collaboration, version control, and team-based access management, it's the perfect solution for technical documentation.

Real-time Collaboration

Work together with your team. See changes as they happen with live editing.

Team Management

Organize members into teams with role-based access control.

Version Control

Full history with automatic backups. Roll back anytime.

Audit Logs

Track every change with detailed audit logs (Team+ plans).

Quick Start

Get started with Solara Docs in just a few steps:

  1. Create an account - Sign up for free or use Google OAuth
  2. Create a project - Set up your first documentation project
  3. Create teams - Organize your collaborators into teams
  4. Add documents - Start writing your documentation
  5. Invite collaborators - Add team members to contribute
Pro Tip
Start with the Free plan to explore all features. You can upgrade anytime as your needs grow.

Projects

Projects are the top-level containers for your documentation. Each project can contain multiple teams, documents, and collaborators.

Creating a Project

Navigate to your dashboard and click "New Project". Enter a name and optionally enable backups (available on paid plans).

Project Limits

Each plan has different limits:

  • Free - 1 project, 2 documents, 3 members
  • Student - 3 projects, 5 documents, 6 members
  • Team - 5 projects, 20 documents, 20 members
  • Enterprise - Unlimited everything

Documents

Documents are where your content lives. Each document belongs to a team within a project, controlling who can view and edit it.

Rich Text Editor

Our editor supports rich text formatting including headers, lists, code blocks, and more. Changes are saved automatically as you type.

Pending Edits

On Team+ plans, editors without direct-save permission will have their changes submitted for review. Admins can approve or reject pending edits.

Teams

Teams help organize your collaborators and control document access. Each team can have multiple members with different roles.

Team Roles

  • Editor - Can edit documents assigned to their team
  • Admin - Can edit + manage documents and approve pending edits

Direct Save Permission

Editors can be granted "direct save" permission, allowing their changes to be saved immediately without admin review.

Collaboration

Solara Docs makes it easy to collaborate with your team on documentation.

Project Roles

  • Owner - Full control over the project
  • Admin - Can manage teams and documents
  • Editor - Can edit assigned documents
  • Viewer - Read-only access

Inviting Collaborators

Project owners can invite users by their username. Invited users become contributors and can be assigned to teams.

Note
Users must have a Solara Docs account before they can be added as collaborators.
